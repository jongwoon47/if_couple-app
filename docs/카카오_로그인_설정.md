# 카카오 로그인 설정 가이드

IF App에서 카카오 로그인을 사용하려면 **카카오 개발자 콘솔** 설정과 **Firebase Secret Manager** 설정을 해야 합니다.

---

## 1. 카카오 개발자 콘솔 설정

### 1-1. 앱 만들기

1. [카카오 개발자 콘솔](https://developers.kakao.com) 접속 후 로그인
2. **내 애플리케이션** → **애플리케이션 추가하기**
3. 앱 이름(예: IF App) 입력 후 저장

### 1-2. REST API 키 확인

1. 만든 앱 선택 → **앱 키** 메뉴
2. **REST API 키** 값을 복사해 두기 (나중에 Firebase Secret에 넣음)

### 1-3. 카카오 로그인 활성화

1. **제품 설정** → **카카오 로그인** → **활성화 설정**
2. **카카오 로그인**을 **ON**으로 설정

### 1-4. Redirect URI 등록

카카오가 인증 후 돌아올 주소를 등록해야 합니다.  
**Firebase Cloud Functions**의 콜백 URL을 넣습니다.

1. **제품 설정** → **카카오 로그인** → **플랫폼** → **Web**
2. **Redirect URI**에 아래 형식으로 추가 (한 줄에 하나씩):

   ```
   https://asia-northeast3-프로젝트ID.cloudfunctions.net/authKakaoCallback
   ```

   - `프로젝트ID`는 Firebase Console에서 확인한 프로젝트 ID
   - 리전이 `asia-northeast3`가 아니면 본인 Functions 리전에 맞게 변경 (예: `us-central1`)

3. **저장** 클릭

### 1-5. 동의 항목 (선택)

- **제품 설정** → **카카오 로그인** → **동의항목**
- 필수 동의항목만으로도 로그인 가능
- 닉네임/프로필 사진 등이 필요하면 해당 항목 활성화

### 1-6. 클라이언트 시크릿 (보안 강화, 선택)

- **앱 키** → **REST API 키** 옆 **코드** 클릭
- **클라이언트 시크릿**을 **활성화**한 뒤 **코드** 생성
- 이 코드를 Firebase Secret `KAKAO_CLIENT_SECRET`에 넣으면 됨 (없어도 동작함)

---

## 2. Firebase Secret Manager 설정

Firebase CLI로 시크릿을 등록합니다.

### 2-1. REST API 키 등록

```powershell
cd c:\dev\couple_app\if_app
firebase functions:secrets:set KAKAO_REST_API_KEY
```

프롬프트가 나오면 **카카오 개발자 콘솔에서 복사한 REST API 키**를 붙여넣고 엔터.

### 2-2. 클라이언트 시크릿 등록 (사용하는 경우만)

```powershell
firebase functions:secrets:set KAKAO_CLIENT_SECRET
```

카카오에서 발급한 **클라이언트 시크릿 코드**를 붙여넣습니다.

---

## 3. Cloud Functions 배포

```powershell
cd c:\dev\couple_app\if_app
firebase deploy --only functions
```

배포가 끝나면 다음 함수들이 생성됩니다.

- `authKakaoStart` – 카카오 로그인 시작 (리다이렉트)
- `authKakaoCallback` – 카카오 인증 콜백 (토큰 발급 후 앱으로 리다이렉트)
- `authKakaoExchange` – 인가 코드로 Firebase Custom Token 교환 (필요 시)

---

## 4. 앱 실행 시 AUTH_API_BASE_URL 지정

웹에서 카카오 로그인을 쓰려면 **인증 서버 주소**를 알려줘야 합니다.  
Firebase Functions 주소를 사용합니다.

### 4-1. Functions URL 확인

Firebase Console → **Functions** → 각 함수 선택 후 **트리거** 탭에서 URL 확인  
또는 배포 로그에 나오는 URL을 사용합니다.

형식 예:

```
https://asia-northeast3-프로젝트ID.cloudfunctions.net
```

**경로 없이** 위와 같은 **도메인만** 넣으면 됩니다.  
앱이 내부적으로 `/authKakaoStart`, `/authKakaoCallback` 등을 붙입니다.

### 4-2. Flutter 웹 실행

```powershell
flutter run -d chrome --web-port=57297 ^
  --dart-define=AUTH_API_BASE_URL=https://asia-northeast3-프로젝트ID.cloudfunctions.net
```

- `프로젝트ID`를 본인 Firebase 프로젝트 ID로 바꾸세요.
- 번역까지 쓰려면 `TRANSLATE_API_URL`도 같이 넣어도 됩니다.

---

## 5. 동작 흐름 요약

1. 사용자가 **카카오 로그인** 버튼 클릭
2. 앱이 `AUTH_API_BASE_URL/authKakaoStart?redirect_uri=앱URL` 로 이동
3. Functions가 카카오 인증 페이지로 리다이렉트
4. 사용자가 카카오에서 로그인/동의
5. 카카오가 `.../authKakaoCallback?code=...&state=...` 로 리다이렉트
6. Functions가 code로 카카오 토큰 발급 → Firebase Custom Token 생성
7. 앱 URL로 `?firebase_custom_token=...` 와 함께 리다이렉트
8. 앱이 `firebase_custom_token`으로 Firebase 로그인 처리

---

## 6. 문제 해결

| 증상 | 확인할 것 |
|------|------------|
| "KAKAO_REST_API_KEY is not set" | Secret Manager에 `KAKAO_REST_API_KEY` 등록 후 Functions 재배포 |
| "redirect_uri mismatch" | 카카오 개발자 콘솔 **Redirect URI**에 `.../authKakaoCallback` 정확히 등록 (프로젝트ID, 리전 확인) |
| "AUTH_API_BASE_URL is not configured" | `flutter run` 시 `--dart-define=AUTH_API_BASE_URL=...` 지정 |
| 로그인 후 흰 화면 | 웹 앱 URL이 카카오 Redirect URI가 아니라, **앱이 열려 있는 URL**이 redirect_uri로 전달되는지 확인 (같은 도메인/포트) |

## 7. 카카오 로그인 버튼 이미지

로그인 화면에서 카카오 공식 버튼 이미지를 쓰려면:

1. [카카오 리소스 다운로드](https://developers.kakao.com/tool/resource/login) 에서 **카카오 로그인 버튼** 이미지 다운로드
2. `assets/images/` 폴더에 `kakao_login_medium_wide.png` 이름으로 저장

이미지가 없어도 노란색 **카카오 로그인** 폴백 버튼으로 동작합니다.
